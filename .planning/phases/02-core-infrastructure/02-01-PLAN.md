```lua
local Bus = {}
Bus.__index = Bus

function Bus.new()
  -- queue: array of {event_name, data}
  -- handlers: table of event_name -> array of handler functions
  -- flushing: boolean re-entrancy guard
  return setmetatable({
    _queue = {},
    _handlers = {},
    _flushing = false,
  }, Bus)
end

function Bus:on(event, handler) ... end
function Bus:off(event, handler) ... end
function Bus:emit(event, data) ... end
function Bus:flush() ... end
```

Key implementation details per CONTEXT.md decisions:
- **Deferred dispatch:** `emit` pushes `{event, data}` onto `_queue`. No handlers called.
- **Flush:** Iterates `_queue`, for each event calls all registered handlers. Clears queue after.
- **Re-entrancy guard:** Set `_flushing = true` during flush. If `emit` is called while `_flushing`, log warning with event name and return without queuing. Use `print()` for logging (no logging framework yet).
- **Error handling:** Wrap each handler call in `pcall`. On error, log with `print()` including error message and stack trace (from `debug.traceback`). Continue to next handler.
- **No priority levels** — registration order only.
- **No wildcard subscriptions** — exact event name matching only.
