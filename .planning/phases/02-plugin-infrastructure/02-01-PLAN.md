Place this BEFORE the component usage comment block (before "-- 1. Component usage"). The variable is intentionally unused (prefixed with underscore) to demonstrate the read pattern without side effects. Keep deps = {} empty per user decision (no fake dependency consumption).

2. **Plugin harness (tests/helpers/plugin_harness.lua):** Add a local `make_dep_enforced_services` function that creates a metatable proxy wrapping real_services. The proxy:
   - Intercepts `get` calls and checks the requested name against an `allowed` set built from `opts.allowed_deps`
   - In strict mode (default): `error("Plugin accessed undeclared service 'X' -- add it to deps", 2)`
   - In tolerant mode: prints warning `[Harness] Plugin accessed undeclared service 'X' -- add it to deps` and falls through to real_services:get()
   - Delegates ALL other method calls (register, etc.) transparently to the real services object via __index
   - Uses method-call syntax wrapping: for non-get function keys, returns a wrapper that calls `v(real_services, ...)`

In `plugin_harness.create_context()`, after the existing deps registration block, add: if `opts.allowed_deps` is provided, resolve error_mode from `(opts.error_mode or "strict")` and replace `ctx.services` with `make_dep_enforced_services(ctx.services, opts.allowed_deps, error_mode)`.

Note: Do NOT use resolve_error_mode from bus/registry — the harness is test-only code. A simple inline default is cleaner here.
