<task type="auto" tdd="true">
  <name>Task 1: Add parse_declared_deps helper and detect_undeclared_service_deps function</name>
  <files>scripts/validate_architecture.lua</files>
  <behavior>
    - parse_declared_deps(lines with "MyPlugin.deps = { 'a', 'b' }") returns {"a", "b"}
    - parse_declared_deps(lines with "MyPlugin.deps = {}") returns {} (empty array)
    - parse_declared_deps(lines with no deps declaration) returns nil
    - parse_declared_deps handles both single and double quotes
    - parse_declared_deps handles trailing commas: { "a", "b", }
    - detect_undeclared_service_deps(plugin_dir) returns errors for undeclared services:get() calls
    - detect_undeclared_service_deps(plugin_dir) returns dep_parse_errors when deps line is missing
    - detect_undeclared_service_deps scans all .lua files under plugin_dir, not just init.lua
  </behavior>
  <action>
1. **parse_declared_deps(lines) [local helper]:** Add as a local function in validate_architecture.lua, placed in the detection section after detect_raw_ecs_calls. It:
   - Iterates lines looking for a match: `^%s*[%a_][%w_]*%.deps%s*=%s*{(.-)}`
   - Extracts the body between `{` and `}`
   - Uses `gmatch` with `["\']([%a_][%w_]*)["\']` to pull all quoted strings from body
   - Returns array of dep names, or nil if no matching line found
   - Handles: empty deps `{}` (returns empty table), trailing commas, mixed quote styles

2. **Validator.detect_undeclared_service_deps(plugin_dir) [module function]:** Add after parse_declared_deps. It:
   - Reads init.lua at `plugin_dir .. "/init.lua"`. If not found, returns empty errors + dep_parse_error `"init.lua not found -- cannot parse deps"`
   - Calls `parse_declared_deps(init_lines)`. If nil (no parseable declaration), returns dep_parse_error `"No parseable 'Module.deps = { ... }' declaration found -- deps must be on a single line"`
   - Builds `allowed` lookup set from declared deps array
   - Calls `find_lua_files(plugin_dir)` to get ALL .lua files under the plugin
   - For each file, reads lines and scans for services:get() calls using pattern: `[sS]ervices%s*:%s*get%s*%(%s*["\']([%a_][%w_]*)["\']`
   - If matched service name is not in allowed set, adds error: `{ file = path, line_num = i, line = line, service = svc_name, message = "services:get('" .. svc_name .. "') -- not declared in deps" }`
   - Returns `errors, dep_parse_errors` (two tables)

Note: The function takes a plugin_dir (e.g., "src/plugins/movement"), NOT individual file paths. This is a different signature than other detect_* functions because it needs to cross-reference init.lua deps against all files in the directory.
  </action>

  </verify>
