<task type="auto">
  <name>Task 2: Configure selene linting with custom Love2D std definition</name>
  <files>selene.toml, love2d.yml</files>
  <action>
Create `selene.toml` at project root:
```toml
std = "love2d"

[rules]
global_usage = "deny"
unused_variable = "warn"
shadowing = "warn"
```

Create `love2d.yml` at project root (selene custom std definition). This file whitelists the Love2D globals plus the project-sanctioned globals from CONTEXT.md decisions (world, eventBus, registry):
```yaml
---
base: "lua51"
name: "love2d"
globals:
  love:
    any: true
  world:
    any: true
  eventBus:
    any: true
  registry:
    any: true
```

The `base: "lua51"` inherits all standard Lua 5.1 globals. Only `love`, `world`, `eventBus`, and `registry` are additionally whitelisted per DEV-01.

Verify selene is available: `selene --version`. If not installed, install via cargo: `cargo install selene` or download the binary.
  </action>
  <verify>Run `selene src/ main.lua conf.lua` — should produce zero warnings/errors. Create a temporary test file with an undeclared global (`foo = 42`), run selene on it, confirm it reports a global_usage violation, then delete the test file.</verify>
  <done>selene.toml configured with global_usage=deny. love2d.yml whitelists exactly love, world, eventBus, registry. selene catches undeclared globals.</done>
