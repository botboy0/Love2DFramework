<task type="auto">
  <name>Task 2: Create architecture validator script</name>
  <files>scripts/validate_architecture.lua</files>
  <action>
Create `scripts/validate_architecture.lua` — a Lua script that scans the project for architectural violations. Per DEV-07, it checks for:

1. **Undeclared globals** — Scan src/ files for global assignments not in the whitelist (love, world, eventBus, registry). Use pattern matching: lines containing `^%s*[a-zA-Z_][a-zA-Z0-9_]*%s*=` that aren't `local` declarations and aren't whitelisted.

2. **Cross-plugin imports** — Scan src/plugins/*/**.lua for `require` calls that reference another plugin's path. Pattern: a file in `src/plugins/foo/` requiring `src.plugins.bar` (where bar ~= foo).

3. **Game logic outside ECS systems** — Scan src/client/ and src/server/ for patterns that suggest direct game state manipulation outside ECS:
   - Direct entity creation/modification outside system files
   - love.update/love.draw containing game logic (should delegate to systems)

4. **Test file mirroring** — For every `.lua` file in `src/` (excluding init.lua pattern for now), verify a corresponding `_spec.lua` exists in `tests/`. Report missing test files.

The script should:
- Accept a `--fix` flag for future auto-fix support (prints "auto-fix not yet implemented")
- Print violations grouped by category
- Exit with code 0 if no violations, code 1 if violations found
- Exclude `lib/` from all checks
- Be runnable as `lua scripts/validate_architecture.lua`

Implementation approach: Use pure Lua (lfs not required). Use `io.popen("find ...")` or recursive directory walking to discover files. Parse each file line-by-line for violation patterns.

Important: The validator should be conservative — flag clear violations, not ambiguous patterns. False positives erode trust in the tool.

1. `lua scripts/validate_architecture.lua` — runs without errors, reports zero violations on current clean project
2. Run selene and stylua on the validator script itself — should pass
