    - love.load() passes transport option to Context.new (default: no transport)
    - love.update() flushes bus THEN flushes transport (bus first, transport second)
    - love.update() handles case where _ctx is nil gracefully
    - Transport receive_all + re-emit happens BEFORE bus flush (inbound messages are queued as bus events before flush delivers them)
  </behavior>
  <action>
Modify `main.lua`:

1. Add Transport require at top (for Transport.new when transport = true):
   ```lua
   local Transport = require("src.core.transport")
   ```

2. Add a local config table (can be empty for now — games override):
   ```lua
   local _config = {}  -- Game-specific config; override in conf.lua or here
   ```

3. Update `love.load()`:
   - Create bus with error_mode from config: `Bus.new({ error_mode = resolve_error_mode(_config, "bus", "tolerant"), log = print })`
   - Or simpler: just pass config through and let modules resolve. Actually Bus.new takes opts.error_mode directly. So:
     ```lua
     local bus_error_mode = (_config.error_modes and _config.error_modes.bus) or _config.error_mode or "tolerant"
     local bus = Bus.new({ error_mode = bus_error_mode })
     ```
   - Create worlds: `Worlds.create()` (single-world default — games can pass `{ dual = true }` via config if needed)
   - Create context with transport: `Context.new({ worlds = worlds, bus = bus, config = _config, transport = _config.transport })`
     - If `_config.transport` is nil → NullTransport (default)
     - If `_config.transport` is a Transport instance → use it
   - Create registry with config: `Registry.new({ config = _config })`
   - Rest of plugin loading unchanged.

4. Add `love.quit()`:
   ```lua
   function love.quit()
       if _registry and _ctx then
           _registry:shutdown(_ctx)
       end
   end
   ```

5. Update `love.update(dt)`:
   ```lua
   function love.update(dt)
       if not _ctx then return end
       -- 1. Receive inbound transport messages and queue as bus events
       local messages = _ctx.transport:receive_all()
       for _, msg in ipairs(messages) do
           _ctx.bus:emit(msg.event, msg.data)
       end
       -- 2. Flush bus (delivers all queued events including inbound transport)
       _ctx.bus:flush()
       -- 3. Flush transport (sends outbound networkable events queued by auto-bridge)
       _ctx.transport:flush()
   end
   ```
