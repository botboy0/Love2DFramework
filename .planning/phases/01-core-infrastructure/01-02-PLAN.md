   - `opts.error_mode` — "strict" or "tolerant" (default: "tolerant")
   - For backward compatibility: if `opts` is a function (not a table), treat it as the log function with tolerant mode. This preserves `Bus.new(print)` usage.
2. Store `self._error_mode = opts.error_mode or "tolerant"` on the instance.
3. In `Bus:flush()`, change the handler dispatch loop:
   - **Tolerant mode** (current behavior): `pcall(handler, data)`, log errors, continue to next handler.
   - **Strict mode**: Call handler directly (no pcall). If it errors, ensure `self._flushing = false` is set before the error propagates. Use a pattern like:
     ```lua
     if self._error_mode == "strict" then
         -- No pcall — let errors propagate
         local ok, err = pcall(list[j], data)
         if not ok then
             self._flushing = false
             error(err, 0)
         end
     else
         local ok, err = pcall(list[j], data)
         if not ok then
             self._log("[Bus] Handler error for event '" .. event .. "': " .. tostring(err))
         end
     end
     ```
     Note: We still use pcall in strict mode so we can reset `_flushing` before re-raising. The `error(err, 0)` re-raises cleanly.
4. Update the module docstring to document error_mode.
