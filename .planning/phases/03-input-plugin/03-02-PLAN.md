    - Actions without touch field are ignored
    - on_touch_pressed(id, x, y) marks active_ids for regions whose rect contains the point
    - on_touch_released(id) removes id from all region active_ids sets
    - update() computes frame transitions: _pressed (just went down), _released (just went up), _down (currently held)

    - get_region_defs() returns table of region definitions for debug overlay (Phase 6)

    - init(ctx) translates ctx.config.input to baton format and creates baton player
    - init registers "input" service with is_down, is_pressed, is_released, get_axis, get_active_device, get_touch_points, get_touch_regions
    - update(dt) calls baton update, touch_regions update, detects pressed/released transitions, emits bus events
    - update(dt) detects device changes and emits input:device_changed
    - on_joystick_added(joystick) sets baton joystick and emits input:gamepad_connected
    - on_joystick_removed(joystick) clears baton joystick and emits input:gamepad_disconnected
    - on_touch_pressed(id, x, y) and on_touch_released(id, x, y) forward to touch_regions
    - shutdown() is a no-op stub (contract established)
    - is_down/is_pressed/is_released unify baton and touch state (OR logic)
    - Config translation: { jump = { key = 'space', gamepad = 'a' } } -> { jump = { 'key:space', 'button:a' } }
    - Config handles array bindings: { key = {'space', 'w'} } -> { 'key:space', 'key:w' }

**touch_regions.lua:**

1. Create `src/plugins/input/touch_regions.lua`:
   - `TouchRegions.new(config, opts)` -- config is the input config table, opts for testing (e.g., `{ screen_w, screen_h }` to avoid love.graphics dependency in tests)
   - Parse each action binding: if `binding.touch` exists, store region def with x, y, w, h, unit
   - `_resolve_rect(region)` -- if unit == "pct", multiply by screen dimensions; else return raw px values. Screen dimensions come from `self._get_dimensions()` which defaults to `love.graphics.getDimensions` but can be overridden via opts for testing.
   - `on_touch_pressed(id, x, y)` -- hit test against all regions, add id to matching region's `_active_ids` set
   - `on_touch_released(id)` -- remove id from ALL regions' `_active_ids` (no hit test needed on release -- touch may have moved)
   - `update()` -- for each region: compute `_down = next(_active_ids) ~= nil`, `_pressed = is_down and not was_down`, `_released = not is_down and was_down`
   - `pressed(action)`, `down(action)`, `released(action)` -- return boolean, false if action has no touch region
   - `get_region_defs()` -- return shallow copy of region definitions (for Phase 6 debug overlay)

**init.lua:**

3. Create `src/plugins/input/init.lua`:
   - Follow canonical_plugin.lua pattern exactly
   - `InputPlugin.name = "input"`, `InputPlugin.deps = {}`
   - `InputPlugin:init(ctx)`:
     a. Store `self._bus = ctx.bus`
     b. Read `ctx.config.input` (default to `{}`)
     c. Translate config to baton format:
        - For each action, binding in config:
          - If binding has `key`: wrap string as `{ key }`, iterate keys, prefix each with `"key:"`
          - If binding has `gamepad`: wrap string as `{ gamepad }`, iterate, prefix with `"button:"`
          - If binding has `gamepad_axis`: wrap string, iterate, prefix with `"axis:"`
          - If binding has `sc` (scancode): wrap string, iterate, prefix with `"sc:"`
          - Collect all sources into baton_controls[action] = sources
        - Store action name set in `self._actions`
     d. Create baton player: `baton.new({ controls = baton_controls, deadzone = ctx.config.input_deadzone or 0.2 })`
     e. Create touch regions: `TouchRegions.new(ctx.config.input or {}, opts)` -- opts nil in production (uses love.graphics)
     f. Store `self._last_device = self._player:getActiveDevice()`
     g. Register "input" service via `ctx.services:register("input", { ... })`:
        - `is_down(action)` returns `self:_is_down(action)`
        - `is_pressed(action)` returns `self:_is_pressed(action)`
        - `is_released(action)` returns `self:_is_released(action)`
        - `get_axis(action)` returns `self._player:get(action)`
        - `get_active_device()` returns `self._player:getActiveDevice()`
        - `get_touch_points()` returns `love.touch and love.touch.getTouches() or {}`
        - `get_touch_regions()` returns `self._touch_regions:get_region_defs()`
   - `InputPlugin:update(_dt)`:
     a. `self._player:update()` (baton frame update)
     b. `self._touch_regions:update()` (touch frame transitions)
     c. For each action in `self._actions`:
        - If `self:_is_pressed(action)` then emit `input:action_pressed` with `{ action = action, device = self._player:getActiveDevice() }`
        - If `self:_is_released(action)` then emit `input:action_released` with `{ action = action, device = self._player:getActiveDevice() }`
     d. Device change detection:
        - `local current = self._player:getActiveDevice()`
        - If `current ~= self._last_device` then emit `input:device_changed` with `{ device = current }`, update `self._last_device`
   - `InputPlugin:_is_down(action)` returns `self._player:down(action) or self._touch_regions:down(action)`
   - `InputPlugin:_is_pressed(action)` returns `self._player:pressed(action) or self._touch_regions:pressed(action)`
   - `InputPlugin:_is_released(action)` returns `self._player:released(action) or self._touch_regions:released(action)`
   - `InputPlugin:on_joystick_added(joystick)`:
     - If `joystick:isGamepad()` and `not self._joystick`: set `self._joystick = joystick`, `self._player.config.joystick = joystick`, emit `input:gamepad_connected`
   - `InputPlugin:on_joystick_removed(joystick)`:
     - If `self._joystick == joystick`: clear both, emit `input:gamepad_disconnected`
   - `InputPlugin:on_touch_pressed(id, x, y)`: forward to `self._touch_regions:on_touch_pressed(id, x, y)`
   - `InputPlugin:on_touch_released(id, x, y)`: forward to `self._touch_regions:on_touch_released(id)`
   - `InputPlugin:shutdown(_ctx)`: no-op stub
