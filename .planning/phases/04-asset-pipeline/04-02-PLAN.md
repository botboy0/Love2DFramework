2. **Create `src/plugins/assets/atlas_builder.lua` (TDD GREEN):**
   - `AtlasBuilder.new(opts)`: accepts `{ ta, love_graphics, drawable_wrapper, log, max_size }`. Defaults: `ta = require("lib.TA")`, `love_graphics = love and love.graphics`, `max_size = 4096`, `log = print`, `drawable_wrapper = require("src.plugins.assets.drawable_wrapper")`.
   - `AtlasBuilder:build(groups, loaded_images)`:
     - For each group in `groups`:
       1. Calculate total pixel area: sum of `(w * h)` for each image in the group
       2. If total area > `self._max_size * self._max_size`, split the group's keys into sub-groups that fit within budget. Emit warning via `self._log`.
       3. For each (sub-)group:
          - Create atlas via `self._ta.newDynamicSize(1, 0, 1)` (padding=1, extrude=0, spacing=1)
          - Call `atlas:setMaxSize(self._max_size, self._max_size)`
          - Add each image via `atlas:add(image, key)`
          - Call `atlas:bake("area")`
          - Extract canvas (check for public accessor, fallback to `atlas._canvas`)
          - For each key, call `atlas:getViewport(key)` to get x,y,w,h
          - Create quad via `self._love_graphics.newQuad(x, y, w, h, canvas:getDimensions())`
          - Create `DrawableWrapper.from_atlas(canvas, quad)` for each key
       4. Store results in `self._atlases[group_name]`
     - Return `self._atlases`
   - `AtlasBuilder:get_atlas(group_name)`: returns `self._atlases[group_name]`
   - `AtlasBuilder:get_all_atlases()`: returns `self._atlases`
   - **Auto-split algorithm:** Sort images by area descending. Greedily fill sub-groups until area budget is reached, then start a new sub-group. Sub-groups named `"group_1"`, `"group_2"`, etc.
