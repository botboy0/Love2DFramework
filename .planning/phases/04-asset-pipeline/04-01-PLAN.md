<task type="auto" tdd="true">
  <name>Task 1: Vendor Lily + RTA and implement manifest parser</name>
  <files>lib/lily.lua, lib/TA.lua, src/plugins/assets/manifest.lua, tests/plugins/assets/manifest_spec.lua</files>
  <behavior>
    - Manifest.parse({}) returns empty tables (no assets configured)
    - Manifest.parse(manifest_table) returns an array of load requests, each with { key, path, type, group, extra }
    - Image entries default group to directory name from path (e.g. "assets/sprites/player.png" -> group "sprites")
    - Image entries with explicit group override use that group
    - Image entries with atlas = false are marked standalone (group = nil)
    - Font entries include size in extra field, always standalone (no group)
    - Sound entries include mode in extra field (default "static"), always standalone
    - Unknown type raises an error
    - Manifest.parse returns (load_requests, groups) where groups maps group_name -> { keys }
  </behavior>
  <action>
1. **Vendor Lily:** Download `lily.lua` from https://github.com/MikuAuahDark/lily (raw main branch). Place at `lib/lily.lua`. This is a single-file library.

2. **Vendor Runtime-TextureAtlas:** Download from https://github.com/EngineerSmith/Runtime-TextureAtlas. The library may be a single file or small directory. If single file, place at `lib/TA.lua`. If directory, place at `lib/RuntimeTextureAtlas/` and create `lib/TA.lua` as a re-export (`return require("lib.RuntimeTextureAtlas")`). Check the repo structure to determine which approach.

3. **Create `src/plugins/assets/manifest.lua`:**
   - `Manifest.parse(manifest_table)` takes the `ctx.config.assets.manifest` table
   - Returns `(load_requests, groups)`:
     - `load_requests`: array of `{ key = string, path = string, type = "image"|"font"|"sound", group = string|nil, extra = table|nil }`
     - `groups`: table mapping group_name -> array of keys (only for atlas-eligible images)
   - **Group assignment logic for images:**
     - If entry has `atlas = false` -> standalone, group = nil
     - If entry has explicit `group = "X"` -> use "X"
     - Otherwise, derive from directory: `"assets/sprites/foo.png"` -> group `"sprites"` (second-to-last path segment)
   - Font entries: extra = `{ size = entry.size or 12 }`, group = nil
   - Sound entries: extra = `{ mode = entry.mode or "static" }`, group = nil
   - Error on unknown type with descriptive message

4. **Create `tests/plugins/assets/manifest_spec.lua`:**
   - Write tests FIRST following TDD (RED-GREEN). All behaviors listed above must have test cases.
   - Use standard busted `describe/it/assert` patterns. No Love2D dependencies — manifest.lua is pure Lua.
  </action>
  <verify>
    <automated>busted tests/plugins/assets/manifest_spec.lua</automated>
  </verify>

    - lib/lily.lua exists and is requireable
    - lib/TA.lua exists (or lib/RuntimeTextureAtlas/ with re-export)

<task type="auto" tdd="true">
  <name>Task 2: Implement drawable wrapper with uniform API</name>
  <files>src/plugins/assets/drawable_wrapper.lua, tests/plugins/assets/drawable_wrapper_spec.lua</files>
  <behavior>
    - DrawableWrapper.from_atlas(atlas_texture, quad) creates an atlas-backed wrapper
    - DrawableWrapper.from_standalone(asset) creates a standalone wrapper
    - wrapper:draw(x, y, r, sx, sy) calls the correct love.graphics.draw variant
    - Atlas wrapper:draw passes (texture, quad, x, y, r, sx, sy)
    - Standalone wrapper:draw passes (asset, x, y, r, sx, sy)
    - wrapper:get_texture() returns the atlas texture or standalone asset
    - wrapper:get_quad() returns the quad for atlas, nil for standalone
    - wrapper:get_dimensions() returns width, height regardless of type
    - wrapper:get_type() returns "atlas" or "standalone"
  </behavior>
  <action>
1. **Create `src/plugins/assets/drawable_wrapper.lua`:**
   - Class with `__index` metatable
   - `DrawableWrapper.from_atlas(atlas_texture, quad)`: stores texture and quad, type = "atlas"
   - `DrawableWrapper.from_standalone(asset)`: stores asset, type = "standalone"
   - `DrawableWrapper:draw(x, y, r, sx, sy)`: uses dependency-injected `_draw_fn` (defaults to `love.graphics.draw`). For atlas: `_draw_fn(self._texture, self._quad, x, y, r, sx, sy)`. For standalone: `_draw_fn(self._asset, x, y, r, sx, sy)`.
   - `DrawableWrapper:get_texture()`: returns `self._texture` or `self._asset`
   - `DrawableWrapper:get_quad()`: returns `self._quad` (nil for standalone)
   - `DrawableWrapper:get_dimensions()`: for atlas, uses quad viewport w,h. For standalone, calls `self._asset:getDimensions()` (or injected equivalent).
   - `DrawableWrapper:get_type()`: returns `self._type`
   - **Dependency injection:** Accept optional `opts` table in constructors with `draw_fn` and `get_dimensions_fn` overrides for testability (following touch_regions.lua pattern).
