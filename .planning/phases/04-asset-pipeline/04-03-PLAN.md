      - Atlas-packed images become DrawableWrappers via from_atlas
      - Standalone images become DrawableWrappers via from_standalone
      - All wrappers stored in self._assets[key]
      - If loading_phase enabled: set self._ready = true, emit "asset:ready"
    - assets:get(key) returns DrawableWrapper if loaded
    - assets:get(key) for manifest asset not yet loaded: error in strict, nil/fallback in tolerant
    - assets:get(key) for on-demand asset still loading: returns nil (never errors)
    - assets:get_atlas(group_name) returns raw atlas data { canvas, wrappers }
    - assets:is_ready() returns self._ready
    - shutdown(ctx) calls loader:shutdown()
    - Fallback assets loaded synchronously if configured and error_mode is tolerant
  </behavior>

     - After simulated batch_complete: atlas builder is called, wrappers created, assets:get returns wrapper

2. **Create `src/plugins/assets/init.lua` (TDD GREEN):**
   - Follow the input plugin pattern exactly for structure.
   - **Plugin metadata:**
     ```lua
     AssetPlugin.name = "assets"
     AssetPlugin.deps = {}
     ```
   - **init(ctx):**
     - Read config: `local assets_config = (ctx.config and ctx.config.assets) or {}`
     - Resolve error_mode from config (use resolve_error_mode pattern — check `config.error_modes.assets`, then `config.error_mode`, default "strict")
     - Parse manifest: `local load_requests, groups = Manifest.parse(assets_config.manifest or {})`
     - Create loader: `self._loader = AssetLoader.new({ bus = ctx.bus, error_mode = error_mode, log = print })`
     - Create atlas builder: `self._atlas_builder = AtlasBuilder.new({ log = print, max_size = 4096 })`
     - Store refs: `self._bus = ctx.bus`, `self._groups = groups`, `self._load_requests = load_requests`
     - Initialize state: `self._assets = {}`, `self._ready = false`, `self._loading_phase = assets_config.loading_phase ~= false` (default true)
     - Start manifest load: `self._loader:load_manifest(load_requests)`
     - Subscribe to batch_complete to trigger atlas packing phase:
       ```lua
       ctx.bus:on("asset:batch_complete", function(_data)
         self:_on_batch_complete()
       end)
       ```
     - Load fallback assets synchronously if configured (tolerant mode only)
     - Register service: `ctx.services:register("assets", { get = function(key) ... end, get_atlas = function(group) ... end, is_ready = function() ... end })`
   - **_on_batch_complete():**
     - Get loaded images: `local loaded = self._loader:get_loaded()`
     - Separate atlas-eligible images from standalone:
       - Images in groups -> pass to atlas builder
       - Standalone images -> wrap with DrawableWrapper.from_standalone
     - Call `self._atlas_builder:build(self._groups, atlas_images)`
     - Collect all wrappers into `self._assets` from atlas builder results
     - Wrap standalone assets (fonts, sounds, non-atlas images) with from_standalone
     - If loading_phase: `self._ready = true`, `self._bus:emit("asset:ready", {})`
   - **update(dt):** `self._loader:update()`
   - **shutdown(ctx):** `self._loader:shutdown()`
   - **Service API (plain functions, not methods):**
     - `get(key)`: If key in `self._assets`, return wrapper. If manifest asset and not loaded: strict -> error, tolerant -> return fallback or nil. If on-demand and loading -> return nil.
     - `get_atlas(group_name)`: Return `self._atlas_builder:get_atlas(group_name)` or nil.
     - `is_ready()`: Return `self._ready`.
