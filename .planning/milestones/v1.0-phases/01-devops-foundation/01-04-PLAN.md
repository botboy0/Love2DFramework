  <action>
Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  check:
    name: Lint, Format, Test, Validate
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install Lua
        uses: leafo/gh-actions-lua@v10
        with:
          luaVersion: "5.1"

      - name: Install LuaRocks
        uses: leafo/gh-actions-luarocks@v4

      - name: Install busted
        run: luarocks install busted

      - name: Install selene
        run: |
          SELENE_VERSION="0.27.1"
          curl -L "https://github.com/Kampfkarren/selene/releases/download/${SELENE_VERSION}/selene-${SELENE_VERSION}-linux.zip" -o selene.zip
          unzip selene.zip
          chmod +x selene
          sudo mv selene /usr/local/bin/

      - name: Install stylua
        run: |
          STYLUA_VERSION="0.20.0"
          curl -L "https://github.com/JohnnyMorganz/StyLua/releases/download/v${STYLUA_VERSION}/stylua-linux-x86_64.zip" -o stylua.zip
          unzip stylua.zip
          chmod +x stylua
          sudo mv stylua /usr/local/bin/

      - name: Lint (selene)
        run: selene src/ main.lua conf.lua

      - name: Format check (stylua)
        run: stylua --check src/ main.lua conf.lua

      - name: Tests (busted)
        run: busted

      - name: Architecture validation
        run: lua scripts/validate_architecture.lua
```

Key design decisions:
- Single job with sequential steps (lint -> format -> test -> validate) per CONTEXT.md
- Uses specific pinned versions for selene and stylua for reproducibility
- Triggers on push to main AND pull requests to main
- Each step fails the entire job if it exits non-zero
- The job name "Lint, Format, Test, Validate" is the required status check for branch protection

Note: Branch protection must be configured manually on GitHub (Settings -> Branches -> Add rule -> Require status checks: "Lint, Format, Test, Validate"). This is a manual step the user must do — document it but do not attempt to automate it.
